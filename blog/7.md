## 正文

CodeWhale（旧版本名为DeepSeek-TUI）它在今年1月由美国独立开发者*Hunter Bown*发起，用Rust语言编写，MIT协议开源，但一直不温不火，直到DeepSeek-V4的发布和Hunter的中文宣传，这个项目在这个五一假期开始爆火。

详细可见[“DeepSeek版Claude Code”，Github 2.3k星](https://www.qbitai.com/2026/05/412914.html)

整个项目围绕着deepseek制作，对deepseek进行了专门适配，这也是我们今天使用它的原因。

### deepseek API
既然要用deepseek+agent，那最重要的当然是deepseek啦。和网页端不同，API入口[在这](https://platform.deepseek.com/api_keys)，打开后先充点钱（不用多少，deepseek的api很便宜，可以先充点尝尝），然后创建一个api key

### 安装Windows Terminal

**liunx用户可跳过**

首先，CodeWhale在终端运行，如果在老旧的cmd或者powershell终端运行可能会出各种问题，因此，我强烈建议使用`Windows Terminal`来运行CodeWhale。这是微软官方推出的现代终端，可以完美避免在老式命令提示符或PowerShell中可能出现的界面闪烁或显示错误等问题。Windows Terminal安装也很简单，直接在微软商店搜索**Windows Terminal**安装即可

### 安装CodeWhale

官方给出了以下安装方法

#### npm安装（推荐）

此方法需要Node.js环境（需Node 18+），如果没有请提前准备好node.js环境

```shell
npm install -g codewhale
```
完成后输入以下命令验证
```shell
codewhale --version
```
如果网络环境不好，请尝试全局科学上网，或者使用以下镜像源安装
```shell
npm install -g codewhale --registry=https://registry.npmmirror.com
```

#### cargo安装（需rust1.88+）

```shell
cargo install codewhale-cli --locked
cargo install codewhale-tui --locked
```
完成后输入以下命令验证
```shell
codewhale --version
```
如果你为Linux 用户并用Cargo编译，需先安装构建依赖（Debian/Ubuntu）
```shell
sudo apt-get install -y build-essential pkg-config libdbus-1-dev
```

除此之外，还有其他安装方式，比如但不限于
- Docker
- Nix
- Windows NSIS 安装包

按需选择，这里不赘述

### 一些小技巧

输入`/mode`可以调整模式

![屏幕截图 2026-06-19 142901.png](https://xundei.dpdns.org/file/7/1781850595610_屏幕截图_2026-06-19_142901.png)

- Plan是只读探索，先给你出个方案再说；
- Agent是默认档，每步工具调用都要你点头；
- YOLO顾名思义，全自动放行，不想被打断就开它。会话可以保存恢复，工作区有独立Git快照兜底，按轮次回滚不动原仓库，直接`/restore`翻车了也不慌。

输入/model可以查看模型

![屏幕截图 2026-06-19 150339.png](https://xundei.dpdns.org/file/7/1781852646103_屏幕截图_2026-06-19_150339.png)

这里可以切换成V4 flash或者pro（deepseek目前API只有这两个模型，其他带deepseek前缀的是旧模型，现在会被重定向到flash或pro）

右侧可以调节思考强度，high够用，但如果你的项目太难了可以调成max
